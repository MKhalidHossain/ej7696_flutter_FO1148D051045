# Support

## Getting help

- Use GitHub issues for reproducible bugs and clearly scoped feature requests.
- For code contributions, review `CONTRIBUTING.md` before opening a pull request.
- For sensitive security concerns, follow `SECURITY.md` instead of creating a public issue.

## Good support requests

A strong support request includes:

- the exact screen or flow involved
- expected behavior versus actual behavior
- reproduction steps
- device, OS, and Flutter version details
- screenshots or logs when relevant

## Project context

This repository contains a Flutter application for inspector exam preparation, resource access, practice sessions, referrals, and payment-enabled unlock flows. Support requests that include the affected feature area are resolved faster.
